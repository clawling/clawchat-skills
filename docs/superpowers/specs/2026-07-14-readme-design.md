# Bilingual README Design

## Goal

Add a conventional GitHub project README that introduces the skills in this
repository, provides English as the default language, offers an in-page link to
Chinese content, and makes the official website and Discord community easy to
find.

## Structure

The repository will use one `README.md` file with:

1. A project title and concise English description.
2. Website and Discord badges, with Discord presented as the community-help
   destination.
3. A language link that jumps to the Chinese section.
4. An English overview and a list of the skills currently present in the
   repository.
5. Website and community links.
6. A complete Chinese section that mirrors the English content and links back
   to English.

## Content Boundaries

- Describe only capabilities confirmed by the existing `SKILL.md` files.
- Include the official website at `https://clawling.com`.
- Include the official Discord at `https://discord.gg/qrfNqTFaG`.
- Do not invent installation commands, compatibility guarantees, or roadmap
  claims that are not documented in the repository.
- Keep the document concise and suitable for the GitHub repository landing
  page.

## Verification

- Confirm the two current skills are represented accurately.
- Confirm the English and Chinese sections are both present and mutually
  linked.
- Confirm website and Discord URLs appear correctly.
- Review Markdown headings, badges, and links for valid syntax.
